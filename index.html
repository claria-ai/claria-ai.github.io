Who is Claria for?

Claria is built for psychologists and clinical practitioners who write evaluation reports. If your day involves psychological evaluations, behavioral assessments, or progress reports, Claria was designed with your workflow in mind.

This is not a generic AI tool. Claria understands the specific instruments you use and generates structured clinical reports that match your practice.

What Claria is not

  • Not a SaaS — nothing is hosted for you. The app runs on your machine.
  • Not a middleman — Claria's developers never see your data.
  • Not a chatbot — it produces structured clinical reports, not freeform conversation.
  • Not a subscription — you pay AWS directly for what you use. No recurring Claria fee.

What is AWS?

Amazon Web Services is a vast constellation of paid computing services. Think of it like a supermarket or home improvement superstore. The products inside the store are varied, but whatever you choose to use is tallied up and billed altogether. The services run in data centers physically located all over the world and you get full control over where data is stored and how it is secured.

Your AWS account is where you turn on services, configure, and use them. In general AWS is treated as a very technical product but Claria aims to make configuration and use straightforward. We don't aim to "hide" AWS — the AWS account used by your Claria Desktop App is always yours. The developers of Claria never know how you're using the app. We gather no telemetry or usage stats. We don't even track the number of downloads.

AWS Bedrock

AWS Bedrock is the key service powering Claria. Released in 2023, Bedrock makes it possible to run a variety of AI systems. The task of inference is complex and expensive: the models are very large and require many computers to run.

Bedrock is metered by token. You are essentially paying "wholesale" for AI usage with no minimums — all while in a HIPAA-approved environment. Anthropic has excellent models that excel at document summary and overall writing, so you're getting the state of the art.

Why not use Anthropic's service directly?

Anthropic requires a certain amount of spend to create a HIPAA-compliant environment. Most practitioners are just dipping their toes into AI and want a safe way to handle patient data while reaping all the benefits of the AI boom.

Claria makes that possible: only pay for what you need and keep everything secure.

macOS users: opening Claria during the beta

Claria is currently ad-hoc signed (no Apple Developer certificate). macOS quarantines apps downloaded from the internet, so you may see a warning that the app "can't be opened." This is a one-time fix.

Before opening the DMG, strip the quarantine attribute in Terminal:

xattr -d com.apple.quarantine ~/Downloads/Claria_0.1.1_aarch64.dmg

Then open the DMG and drag Claria to Applications. If you already copied it:

xattr -cr /Applications/Claria.app

Proper Apple Developer signing and notarization is planned — this workaround will go away in a future release.

Getting started

Once you install Claria, the app walks you through everything. You don't need to be technical — the built-in setup wizard handles AWS configuration, security settings, and HIPAA compliance for you.

  1. Install Claria and create an AWS account

    • The setup wizard guides you through creating your AWS account (~10 minutes)
    • You'll need an email address and a credit card
    • Claria configures encryption, access controls, and storage automatically

  2. Secure your environment

    • Claria walks you through signing the AWS Business Associate Agreement for HIPAA
    • Enables encryption on all data at rest and in transit
    • Provisions a private S3 bucket that only your account can access

  3. Upload your data to S3

    • Configure global inference rules and templates
    • Start a patient record with intake forms (PDF, scans, etc.)
    • Add patient-specific inference rules and templates

  4. Generate reports

    • Select assessments and instruments
    • Review the cost estimate before anything runs
    • Generate, preview, edit, and export to DOCX or PDF

Technical details

  • Data is never sent to Claria's developers — the app is an interface to AWS services
  • You are always the owner of your AWS account
  • Patient data is stored encrypted in AWS S3
  • Built in Rust for memory safety and performance, using the official AWS SDK for Rust
  • 9 focused, auditable crates — each one has a single responsibility
  • No telemetry, no usage stats, no download tracking
  • Everything is open source, including how we build the downloadable release
